Exactly. So the features are:

    Check — is the service running or not
    Restart — if it's down, restart it automatically
    Log the incident — write to a file when it goes down and when it comes back up
    Crash history — read that log file and show the user how many times it died
    Interval — check every N seconds in a loop

That maps to these functions:

    check_service(name) — returns True/False if service is running
    restart_service(name) — restarts it
    log_incident(name, event) — writes crash/recovery to a log file
    show_history(name) — reads the log file and prints crash history
    watch(name, interval) — the main loop that ties it all together
